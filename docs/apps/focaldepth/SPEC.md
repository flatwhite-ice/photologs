# FocalDepth — 스펙

> 사진가용 **카메라 포맷 환산기**. 임의의 필름/센서 포맷 + 렌즈 초점거리 + 조리개를
> 입력하면 **135 풀프레임(36×24mm) 기준**의 환산 화각·환산 조리개(심도 등가)와
> 실제 심도를 실시간 계산한다.

- **상태**: v0.1 (초기 구현 완료, 실브라우저 검증 완료)
- **최종 갱신**: 2026-06-03
- **위치**: `docs/apps/focaldepth/` (photologs 사이트 내 독립 정적 앱)
- **라이선스/배포**: GitHub Pages 정적 발행, 빌드 없음

---

## 1. 목적 / 배경
- 다양한 포맷(대형·중형 필름, APS, 1인치, 아이폰 등)에서 동일 렌즈가 만드는 화각·심도를
  사진가에게 익숙한 **풀프레임 기준**으로 직관적으로 보여준다.
- 대표 질문: *“6×12 포맷에서 58mm f/5.6 → FF 환산은?”* → **20mm f/1.9**.
- 추적·개선 가능하도록 스펙을 분리 관리하며, 향후 독립 저장소로 분리할 수 있게 설계한다.

## 3. 환산 수학 (검증 완료)
기준: 135 FF = 36×24mm, 대각 `43.267mm`.

| 항목 | 공식 |
| --- | --- |
| 포맷 대각 | `diag = √(w² + h²)` |
| 크롭 팩터 | `crop = 43.267 / diag` (대형·중형 <1, 크롭·폰 >1) |
| 환산 초점거리 | `equivFocal = focal × crop` (대각 기준 관행) |
| 환산 조리개(심도·배경흐림 등가) | `equivAperture = fNumber × crop` |
| 화각(도) | `aov(d, f) = 2·atan(d / 2f)` — 장변/단변/대각 각각 |
| 착란원(CoC) | `c = diag / 1500` (Zeiss 관행) |
| 과초점거리 | `H = f²/(N·c) + f` (mm) |
| 근점 / 원점 | `Dn = H·s/(H+(s−f))`, `Df = H·s/(H−(s−f))`, `H ≤ s−f` 이면 `Df = ∞` (s = 초점거리 mm) |

**검증 앵커** (`node` 단위 점검):
- 6×12 (56×112) / 58mm f/5.6 → crop 0.346, **20.0mm · f/1.9** ✔
- FF 50mm → crop 1.00 ✔ · APS-C → 1.53 ✔ · 645 → 0.62 ✔ · 8×10 → 0.14 ✔
- iPhone 15 Pro 메인 → 9.76×7.32(≈1/1.28″), crop 3.55, 환산 24mm ✔
- 실제 심도 FF 50mm f/8 @3m → 과초점 10.9m, 근점 2.36m, 원점 4.12m ✔

## 4. 데이터시트 (`src/formats.js`)
엔트리: `{ id, name, w, h(mm), note?, est?, focal?, fnumber?, equiv?, ref? }`.
`w,h`(노광 이미지 영역)가 모든 계산의 단일 출처. 폰은 네이티브 `focal/fnumber/equiv`를
가져 선택 시 입력값을 자동 프리필한다.

**카테고리 (총 86 엔트리):**
- **소형(135)**: 풀프레임 36×24 (기준).
- **APS / 1인치**: APS-C 소니·니콘·후지 23.5×15.6, APS-C 캐논 22.3×14.9, APS-H 28.7×19.0, 1인치 13.2×8.8.
- **중형(120)**: 단변 56mm — 645·6×6·6×7·6×8·6×9·6×12·6×17·6×24.
- **중형 디지털백**: 44×33(43.8×32.9, GFX·핫셀블라드 X/CFV-50c·II 50C·645Z), 49×36.7(CCD 백: CFV-39·CFV-50 CCD), 645 디지털 풀프레임 53.4×40(Phase One IQ4·H6D-100c·X2D).
- **대형(시트, 이미지 영역)**: 4×5·5×7·8×10·11×14 + 파노라마 4×10·5×12·7×17·8×20.
- **아이폰(3G ~ 17 Pro/Max)**: 단일/멀티(초광각·광각·망원) 카메라별. `est:true`.

**아이폰 추정 방식**: 애플이 센서 실치수를 공개하지 않으므로, 공개된
**35mm 환산 초점거리(equiv)와 실초점거리(focal)에서 크롭을 역산** →
`diag = 43.267/crop`, 4:3 가정으로 `w=0.8·diag, h=0.6·diag`. 화각·심도 결과는
알려진 환산값과 일관된다. 동일 세대 공통 하드웨어는 한 엔트리로 묶음(예: "12 / mini").

**출처**: Apple 기술 사양·EXIF 환산 초점거리, DPReview·Wikipedia 분해 추정치,
필름 규격(120/시트필름). `FORMAT_NOTES` 배열 → UI "데이터 출처·주석"에 노출.

## 5. UI 동작
- **입력**: 포맷 셀렉트(`<optgroup>`), 초점거리/조리개/(고급)초점거리 — 숫자+슬라이더 양방향 동기화. 조리개 슬라이더는 표준 스톱 인덱스, 직접 입력 시 최근접 스톱으로 스냅. 숫자 입력에 단위 표기(mm / f/ / m), `focus-within` 강조.
- **결과 히어로**: 환산 초점거리·환산 조리개를 풀폭 accent 카드로 강조(가장 먼저 인식되는 영역). 입력 변경 시 **숫자 카운트업 트윈(easeOutCubic ~380ms) + 강조 펄스**(`setStat`/`pulse`, `app.js`). 그 외 크롭·화각·SVG·(고급)실제 심도는 즉시 갱신.
- **모바일(≤760px)**: 입력 패널이 **하단 고정 시트**로 전환(`position:fixed`, 핸들 탭으로 접기/펼치기, 요약 텍스트로 현재값 노출). 결과는 상단에 스티키. 데스크톱은 2단(좌측 sticky) 유지.
- **테마**: 공유 다크모드 토글에 자동 대응(CSS 변수). `prefers-reduced-motion` 시 트윈·펄스·시트 트랜지션 생략.

## 6. 알려진 한계
- 환산 초점거리는 **대각 기준**이라 종횡비가 다른 포맷 간 정확히 같은 프레이밍은 아님(장/단변 화각 별도 표기로 보완).
- 아이폰 등 `est` 항목은 추정치 — 실측 센서와 오차 가능. 망원/구형 일부는 신뢰도 낮음.
- 대형 포맷 이미지 영역은 카메라/홀더에 따라 ±수 mm 차이.
- CoC는 `diag/1500` 고정(사용자 지정 불가).
- 클라이언트 렌더만 — JS 비활성 환경 미지원.

## 8. 독립 저장소 분리 계획
이 폴더는 자체 완결적 단위로 설계됨. 분리 시:
1. `docs/apps/focaldepth/` 전체를 새 저장소 루트로 이동.
2. 공유 의존 3종을 벤더링: `shared/{style.css, theme.js, nav.js}` → 새 저장소 `src/shared/`(또는 `vendor/`)로 복사.
3. `index.html` 경로 치환: `../../shared/…` → 벤더링 위치, `src/…`는 그대로.
4. `nav.js`의 `data-home`을 새 사이트 홈으로 조정(또는 헤더 단순화).
5. 자체 배포 설정(GitHub Pages `/` 또는 `/docs`) + `.nojekyll`.
- 분리 후에도 `convert.js`/`formats.js`는 변경 불필요(순수·자족적).

## 9. 검증 방법
- **엔진 단위**: `node -e` 로 `require('./src/convert.js')` 후 앵커값 점검(§3).
- **DOM 와이어링**: jsdom으로 `index.html`+`src/*` 로드 → 결과 엘리먼트 텍스트 확인.
- **실브라우저**: 저장소 루트에서 `python3 -m http.server 8000` →
  `http://127.0.0.1:8000/docs/apps/focaldepth/`. 라이트/다크 토글·슬라이더·고급 심도·SVG·모바일 폭 확인.
  (이 환경 검증 방법은 저장소 메모리 `browser-verification` 참고: 시스템 Chrome + Playwright `chrome` 채널.)
